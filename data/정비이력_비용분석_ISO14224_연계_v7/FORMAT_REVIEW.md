# 정비이력 데이터 포맷 검토 (xlsx vs PDF)

## 배경
- 데이터: 작업지시서/작업완료보고서 40건 + 공구/장비 품목리스트 1건
- 용도: Qdrant 기반 RAG 검색 + 사람이 읽을 수 있는 정비이력 문서

## 비교 분석

| 항목 | xlsx | PDF |
|------|------|-----|
| RAG 텍스트 추출 | openpyxl 등 라이브러리 필요, 셀 위치 파싱 복잡 | PyMuPDF/pdfplumber 등으로 텍스트 직접 추출 용이 |
| 사람 가독성 | 스프레드시트 앱 필요, 레이아웃 불안정 | 어디서든 동일하게 렌더링, 인쇄 최적 |
| 산업 표준 | 데이터 입력/수정용 | 정비이력 최종 문서로 ISO 9001/14224에서 권장 |
| 검색 임베딩 | 셀 단위 파싱 → 문맥 손실 가능 | 페이지/섹션 단위 청킹으로 문맥 보존 |
| 버전 관리 | 바이너리, diff 불가 | 바이너리, diff 불가 (동일) |
| 데이터 재활용 | 구조화 데이터 직접 접근 가능 | 별도 JSON 파싱 결과 병행 필요 |

## 결론

**PDF를 최종 Knowledge Base 포맷으로 채택한다.**

### 이유
1. **RAG 검색 최적화**: PDF에서 텍스트 추출이 더 안정적이며, 섹션 기반 청킹이 자연스러움
2. **산업 표준 준수**: 정비이력 문서는 PDF가 국제 표준 (ISO 14224, ISO 9001 문서 관리)
3. **가독성**: 현장 담당자가 인쇄/열람 시 일관된 레이아웃 보장
4. **이중 보관**: 원본 xlsx는 데이터 입력 이력으로 보관, PDF는 최종 문서로 Knowledge Base에 적재

### 보완 조치
- 파싱된 구조화 데이터는 `data/parsed_maintenance_records.json`으로 별도 보관
- xlsx 원본은 `data/` 디렉토리에 그대로 유지
- PDF 생성 시 fpdf2 + Noto Sans KR 폰트 사용 (한글 깨짐 방지)
